# 0.7 metrics fixture — exemplar recipe specs (hand-harvested from survey-core v2.5.33 default-theme SCSS)

Spec for the 0.7 implementer: every value is FORMULA-first (recipes compute from resolved tokens; defaults shown at base-unit 8 / font-size 16 / editor-font-size 16 / corner-radius 4). `calcSize(n)` = n × `--sjs-base-unit`; `calcFontSize(n)`/`calcLineHeight(n)` = n × `--sjs-font-size`; editor-font line-height = 1.5 × `--sjs-font-editorfont-size` (SEPARATE token from calcLineHeight(1.5) — identical at defaults, diverges under custom themes; two tokens, never merged).

Web-only constructs intentionally not carried: `appearance` resets, `cursor`, `outline`, CSS `transition` (animation policy 0.4 D3), negative-margin/float gutter hacks (replaced by flex), inset-shadow-as-border (shadows now map natively via boxShadow arrays; borders remain the FOCUS/ERROR ring convention).
