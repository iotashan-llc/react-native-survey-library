## What tagbox is

`tagbox` is the **multi-select** sibling of `dropdown`. Same touch/overlay
machinery (`_setIsTouch(true)` → `dropdownListModel` + `popupModel`), but:

- `question.value` is an **array** (e.g. `["a","b"]`), not a scalar.
- The control shows the selected items as removable **chips**, not a single
  value text.
- The overlay list stays **open** across selections (multi-pick) and shows a
  checkmark per selected row (core's `ListModel.isItemSelected` — the 2.1
  ListPicker already renders this).

Verified against survey-core 2.5.33: `getTemplate() === "tagbox"`,
`dropdownListModel` present, `value` is an array.

## Open questions — RESOLVED (probed against survey-core 2.5.33)

- **Chips source**: `vm.getSelectedActions()` → Actions, each with `.title`
  (display) + `.id` (the item value). Render one chip per action.
- **Multi-select uses the EXISTING `listModel.onItemClick`**: clicking an
  unselected item ADDS to the array (`["a","b"]` → click c → `["a","b","c"]`);
  re-clicking a selected item REMOVES it (click a → `["b","c"]`). Core toggles —
  the 2.1 ListPicker needs NO change. The list stays open (core does not hide
  `popupModel` per-select), so the 2.1 bridge (dismiss-only-on-visibility-event)
  keeps the overlay open for multi-pick with no change.
- **Chip ✕ removal**: `question.value = question.value.filter(v => v !== id)`
  via the PUBLIC value setter (setting value IS the documented API — not
  re-deriving model logic, so invariant 6 holds), OR route through the list
  toggle. `vm.removeLastSelectedItem()` exists for backspace semantics.
- **`renderAs`**: defaults to `"default"`; guard a `"select"` fallback the same
  way 2.3 does (r2 #1) — non-interactive display + diagnostic when no VM.
- **`showOtherItem`**: supported (`hasOther`) → reuse `DropdownOtherComment`.
