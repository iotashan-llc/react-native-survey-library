# Task 1.1 design — `<Survey>` root: json XOR model, pre-model URL preflight, applyTheme, owned-model dispose, SurveyRef, event props (A11, A12)

Status: draft for orchestrator review (CORE task — needs orchestrator/llm-pair sign-off; implementation proceeds TDD-first in parallel per coordinator instruction). Consumes task 1.2's bridge API (skeleton pending on `origin/feat/1.2-lifecycle-bridge`); bridge-dependent wiring is behind an explicit seam until it lands. ## Verified upstream facts the design rests on

- `survey-react-ui`'s `Survey` (reactSurvey.tsx) is the porting reference: `createSurvey` (model passthrough vs `new SurveyModel(json)`), `isModelJSONChanged` via `Helpers.isTwoValueEquals(json, previousJSON)` (deep compare — equal-but-new-reference json must NOT recreate), destroy/recreate on change, `updateSurvey`'s mechanical `on*` prop → `EventBase.add/remove` wiring, `renderCallback = () => setState(bump)`, state switch `completed | completedbefore | loading | empty | running/starting`, page render dispatched through the element factory key `survey.pageComponent || "sv-page"` with props `{survey, page}`.
- Upstream recreates the model inside `shouldComponentUpdate` — a render-phase side effect. NOT ported (violates the 0.4 React-19 contracts); ownership moves to the commit phase (below).
- survey.d.ts (typings/src/survey.d.ts): `applyTheme(theme: ITheme): void` (:3309), `scrollToTopOnPageChange(doScroll?: boolean): void` (:2100, public), `renderCallback: () => void` (:939), `onScrollToTop: EventBase<SurveyModel, ScrollToTopEvent>` (:816). `Helpers.isTwoValueEquals` static (helpers.d.ts:24). `Base` has `dispose()` / `isDisposed`.
- `choicesByUrl` fires its network request at model construction/attach (0.3 ledger; A11 rationale) — the ONLY safe interception is before `new Model(json)`.
- `afterRenderSurvey`/`rootElement` deliberately never called/fabricated (1.2 design, "What the bridge deliberately does NOT do").

## Props (A12)

```ts
export interface SurveyOwnProps {
  /** Untrusted survey JSON — preflighted (A11) before model construction. XOR with `model`. */
  json?: unknown;
  /** Host-constructed model — documented TRUSTED/prevalidated by the host (A11). Never disposed by us. */
  model?: SurveyModel;
  /** ITheme, passed unmodified: fed to BOTH `model.applyTheme` and `SurveyThemeProvider`. */
  theme?: ITheme;
  /** A12 per-component slot overrides — forwarded to `SurveyThemeProvider`. */
  styles?: SurveyComponentStyles;
  /** URI-policy config for the json-path preflight (render-time sinks keep their own config seams per 0.9). */
  uriPolicy?: UriPolicyConfig;
  /** RN-level scroll interception with preventDefault semantics — consumed by the 1.2 bridge (NOT a model event). */
  onScrollToElement?: (event: SurveyScrollToElementEvent) => void;
}
export type SurveyProps = SurveyOwnProps & SurveyModelEventProps;
export interface SurveyRefHandle {
  model: SurveyModel | null;
  focusQuestion(name: string): boolean; // delegates model.focusQuestion → funnel → bridge
  scrollToTop(): void;                  // scroll host to y=0
}
```

- **XOR enforcement (non-throwing, invariant 9):** both provided → `model` wins + diagnostic `survey-root-conflicting-props`; neither → render `null` + diagnostic `survey-root-missing-model` (task 1.8 refines the empty state).
- **No arbitrary model-property passthrough.** Upstream's `updateSurvey` assigns any unknown prop onto the model; we accept ONLY the typed surface above (hosts set model properties on the model). DIFFERENCES.md entry.

### Event props — derived, not hand-listed

`SurveyModelEventProps` is a **mapped type over `SurveyModel`**: every key matching `` `on${string}` `` whose value has an EventBase-shaped `add` accepts an optional prop typed as that event's exact handler. This IS A12's "derived from actual `EventBase` members" — the compiler re-derives the surface from the installed survey-core on every typecheck; upstream drift surfaces as type errors, not silent gaps.

Runtime: `wireModelEventProps(model, prevProps, nextProps)` — for each `on*` key where `model[key]` has `add`/`remove`: identity-diff prev vs next, `remove(prev)` / `add(next)`. Consumer functions are subscribed RAW (remove-by-identity works). Full unwire on model-swap AND unmount — upstream leaks host-model subscriptions on unmount; we do not (0.4 deterministic-unsubscribe ethos; observable via `hasActiveUISubscribers`? no — via `EventBase.isEmpty`). `onScrollToElement` is excluded (RN-level). Consumers may also subscribe `onScrollToTop` directly and will observe events with `allow` already `false` (1.2 "Consumer observability").

## Model ownership

- **json path**: `preflightSurveyJson(json, uriPolicy)` → `new Model(cleanJson)` → owned. Disposed on json-change swap and on unmount.
- **json change detection**: `Helpers.isTwoValueEquals` against the json snapshot, evaluated in the commit phase (`componentDidUpdate`-equivalent), never in render (0.4 contract). One commit renders the old model, the swap commit replaces it — acceptable, matches React lifecycle law.
- **model path**: host-owned. Identity change → full rewire (events unwired from old, wired to new; bridge uninstall/reinstall; NO dispose ever).
- **StrictMode remount simulation**: unmount-simulation disposes an owned model, remount must not render a disposed model — on mount, an owned model with `isDisposed === true` is reconstructed from the retained json snapshot (re-preflighted — cheap, pure). Regression-tested under `<React.StrictMode>`.

## Pre-model URL preflight (A11) — `src/security/json-preflight.ts`

Pure function, no side effects: `preflightSurveyJson(json, config?): { json, diagnostics }`.

- Deep-walks the raw JSON (pages/elements/panels recursion + templateElements) for URL-bearing fields → `validateUri(value, context, config)`:
  | JSON field | context |
  |---|---|
  | `choicesByUrl.url` (and legacy string form `choicesByUrl: "…"`) | `choicesByUrl` |
  | `imageLink` (image question, imagepicker choices) | `image` (`video` when the owning element's `contentMode === 'video'`) |
  | survey `logo` | `image` |
  | survey `backgroundImage` | `background` |
- HTML-bearing fields are NOT preflighted here — the render-time sanitizer + press-time revalidation own those (0.9); `link` context is human-mediated by definition.
- **Fail closed by stripping on a clone**: offending property removed from a lazily-created structural clone (the consumer's object is NEVER mutated — "JSON unmodified" contract); one diagnostic per strip: `survey-json-blocked-url` `{path, context, reason}`. No offenders → the ORIGINAL reference is returned (zero-cost hot path).
- Valid URLs pass through **as authored** (not canonical-rewritten): for `choicesByUrl` the fetch sink is survey-core itself, which cannot be handed our canonical string; render-time sinks we own re-validate and consume canonical per 0.9 (defense in depth).
- **Documented residual gap** (DIFFERENCES.md): core-owned `choicesByUrl` fetches can't enforce the manual-redirect rule (`requiresManualRedirect`) from outside; the empirical abort gate is task 2.3's deliverable. Preflight guarantees scheme/origin policy only.

## Theme

`theme` prop present → (a) `model.applyTheme(theme)` on model creation/swap and on theme change (provider-style canonical snapshot compare, not reference compare), keeping model-side derived state (css vars, headerView, isPanelless, backgroundImage merge) consistent; (b) same object to `<SurveyThemeProvider theme styles>`. No `theme` → no `applyTheme` call, provider defaults. Applies to host-owned models too — passing `theme` means "theme this survey" (documented).

## Responsive ownership (0.7 provider contract)

The provider doc assigns this to the Survey root explicitly: root
`onLayout` → `width < 600` → (a) the provider's `narrow` prop and (b)
`survey.setIsMobile(narrow)`. Implemented in 1.1: the inner root's
container `onLayout` reports narrow-transitions up; the outer applies
both sinks (guarded per (model, narrow) pair, so remounts/StrictMode
never double-call).

## Render body (v0.1 shell)

```
<SurveyThemeProvider theme={theme} styles={styles} narrow={narrow}>
  <LifecycleContext.Provider value={{registry}}>  ← 1.2's context
    <View onLayout={narrow detection}>            ← root
      <ScrollView ref onScroll onLayout>          ← registered as the 1.2 scroll host; onScroll+onLayout keep getViewport() answerable (scrollIfVisible)
        {running/starting → RNElementFactory.createElement(survey.pageComponent || 'sv-page', { survey, page: survey.activePage })}
        {completed/completedbefore/loading/empty → protected seam methods, null for now (task 1.8 fills)}
      </ScrollView>
    </View>
  </LifecycleProvider>
</SurveyThemeProvider>
```

- `sv-page` stays unregistered until task 1.4 → `createElement` returns `null` → shell renders an empty scroll region without crashing; tests register a stub to assert the dispatch key + props contract.
- The `normalizedBackground` layer is DEFERRED to the 1.4/1.8 shell-composition work (0.7's ownership fact stands — the root will "just render" the provider's normalized value; it is simply not part of this task's plan row).
- Inner `SurveyRoot` is a class extending `SurveyElementBase` (`getStateElement() → survey`) — model property changes (currentPage, state) re-render through the 0.4 mechanism; `renderCallback` additionally ported as a `__svRev`-style bump (upstream parity for core-initiated full re-renders).
- Outer `Survey` (public export) is a thin `forwardRef` function component owning model lifecycle (preflight/construct/dispose/swap), theme application, and the imperative `SurveyRefHandle`. Hooks here do not touch the reactive-binding mechanism (A3 governs model→setState binding, which stays in the class).

## Bridge wiring (1.2 seam)

On mount (and per model, via the `key` remount): create registry,
`installLifecycleBridge(survey, registry, {})`; uninstall + scroll-host
deregister on unmount. ScrollView ref wrapped as `ScrollHostHandle
{scrollTo, measureTarget, getViewport}`; `onScroll` + `onLayout` keep
`getViewport()` answerable (bridge-author note: that is how
`scrollIfVisible` is honored; `null` before first layout = unconditional
scroll, degraded but safe). `componentDidUpdate` detecting `activePage`
change while `running`/`starting` fires the render-complete call (1.2
"Sequencing") mirroring core's `afterRenderPage` (survey.ts:5514-5519)
EXACTLY — either/or, never both: a pending `survey.focusingQuestionInfo`
routes to `focusQuestionInfo()` (private; called via cast) and SUPPRESSES
the page-change scroll; otherwise `survey.scrollToTopOnPageChange()` →
re-enters the funnel → bridge scrolls/focuses.

**Open item (flagged to the 1.2 task):** `LifecycleBridgeOptions` has no
consult seam for the consumer-facing `onScrollToElement` preventDefault
semantics the 1.2 design assigns to the bridge. The prop is accepted (so
the A12 surface is stable) but wiring waits for an
`onScrollRequest`-style option; suggested semantics: suppression skips
the scroll, focus-intent completion still runs.

## Test plan (TDD; red observed before implementation)

1. **Preflight (pure)**: disallowed `choicesByUrl` stripped on clone + diagnostic + consumer json unmutated; allowed origin passes untouched (same reference when no offenders); string-form `choicesByUrl`; `imageLink`/`logo`/`backgroundImage` contexts; nested panels + paneldynamic `templateElements` walk; `contentMode: 'video'` → video context.
2. **XOR**: both → model wins + diagnostic; neither → null + diagnostic.
3. **Ownership**: json path constructs AFTER preflight (a blocked choicesByUrl never reaches the constructor); unmount disposes owned; host model never disposed; deep-equal-but-new-reference json → no recreation; json content change → old disposed, new live; model-prop swap rewires.
4. **StrictMode**: owned model usable after remount simulation.
5. **Event props**: model event reaches prop handler; handler identity swap → no double-fire, old removed; unmount/model-swap fully unwire — asserted BEHAVIORALLY (handlers never fire afterward), because `EventBase.isEmpty` is not a valid observable: a fresh `Model` already carries internal core subscribers (empirical finding, 2.5.33); onScrollToElement NOT wired to the model.
6. **Theme**: applyTheme called with the prop theme on mount + on theme change, not when absent; provider context reflects the theme (probe via context consumer).
7. **Render dispatch**: stub `sv-page` receives `{survey, page: activePage}`; `survey.pageComponent` override honored; unregistered key renders null, no crash, non-running states render null shell.
8. **SurveyRef**: exposes live model + helpers; null model (neither prop) → `model: null`, helpers no-op.
9. **Bridge wiring** (blocked on 1.2 skeleton): install on mount, uninstall on unmount/swap, scroll host registered, page change → afterRenderPage-parity either/or (pending `focusingQuestionInfo` → `focusQuestionInfo()`, scroll suppressed; else `scrollToTopOnPageChange`) → funnel intercepted natively, `onScrollToElement` preventDefault suppresses scroll.
10. Public export additions covered by the 0.8 api gate + index export test.
