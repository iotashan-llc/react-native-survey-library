# Task 2.8c — paneldynamic tab mode (+ adaptive overflow)

Status: **DRAFT**. Extends 2.8b (stacked on `feat/2.8b-paneldynamic-carousel`).
Weight M. Not CORE — a bounded extension reusing the 2.8a/2.8b patterns.

## What changes vs 2.8b

2.8b handles `list` + `carousel`; `tab` still shows the
`paneldynamic-mode-unsupported` fallback. 2.8c adds the `"tab"` branch.

## Verified model surface (survey-core 2.5.33)

- `displayMode === "tab"` → `isRenderModeTab` true, `renderedPanels` is a SINGLE
  `[currentPanel]` (like carousel), `currentIndex` / `currentPanel`.
- Navigate to a tab: assign `question.currentIndex = i`.
- Tabs = one per `visiblePanels[i]`. A per-panel tab title (`panel.locTitle`) is
  empty by default, so label a tab by its `locTitle.renderedHtml` when present,
  else the 1-based index.
- Reactivity: `currentIndexChangedCallback` (assigned in 2.8a's
  `attachCallbacks`) fires on `currentIndex` change → a tab tap re-renders.

## Design (extend `PanelDynamicQuestion.renderElement`)

- `isRenderModeList` → list (2.8a). `displayMode === "carousel"` → carousel (2.8b).
- else `displayMode === "tab"` → tab:
  - a `paneldynamic-tabs` HORIZONTAL `ScrollView` strip: one
    `paneldynamic-tab-<i>` Pressable per `visiblePanels[i]`
    (`role="tab"`, `accessibilityState.selected` when `i === currentIndex`,
    label = panel title or `i + 1`); tap → `question.currentIndex = i`.
  - the current panel (`renderedPanels[0]`) via the existing `PanelDynamicItem`.
  - the same add button + (via the item) remove.
- else (`progressTop`/`progressBottom`-only display or an unknown mode) → keep
  the unsupported fallback + deferred diagnostic.

### Adaptive overflow → a scrollable strip (documented DIFFERENCE)

survey-core's tab mode adaptively overflows tabs that don't fit into a popup
action container (`isMobile`, responsive measurement). RN has no DOM measurement
loop; the native idiom is a horizontal `ScrollView` that simply scrolls to the
off-screen tabs. 2.8c ships the scrollable strip and documents the
no-overflow-popup delta in DIFFERENCES. (A measured overflow-to-OverlayHost popup
could be a later enhancement, but is out of scope for v1.)

## Test plan (TDD, red first)

- tab mode renders a `paneldynamic-tabs` strip with one tab per panel + the
  single current panel; not list, not carousel.
- the current tab (index 0) is `accessibilityState.selected`; others not.
- tapping tab 2 sets `currentIndex` to 2 and renders that panel
  (renderedPanels[0] id present, others absent).
- add/remove still work in tab mode.
- an unknown/other displayMode still → unsupported fallback + diagnostic.
- list + carousel unchanged (regression, already covered).

## Wiring

None beyond the component. DIFFERENCES: tab strip is a horizontal scroll (no
overflow-to-popup). kitchen-sink: optional; parity regen if added.
