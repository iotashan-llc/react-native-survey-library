## Problem

React Native aliases `window === global`. survey-core's SSR guard is `DomWindowHelper.isAvailable() = typeof window !== "undefined"` (`src/global_variables_utils.ts`), so every window-guard PASSES in RN, then real DOM calls hit RN's global where the API is missing. Verified require-time crash (survey-core 2.5.33): `src/dragdrop/dom-adapter.ts:9-21` top-level module code calls `DomWindowHelper.getWindow().addEventListener("touchmove", …)` → TypeError at `require('survey-core')`, before any consumer code runs.

`document` is NOT defined in RN, so all `DomDocumentHelper` guards take the SSR path (e.g. `settings.ts`'s environment ternary → `undefined`). Document absence gates the require-time surface, but does NOT make all runtime paths safe — see ledger.

## Require-time surface (verified against 2.5.33 source)

| Site | Behavior in RN | Action |
|---|---|---|
| `dragdrop/dom-adapter.ts:9` top-level `window.addEventListener("touchmove",…)` | TypeError, crash at require | shim: no-op `addEventListener`/`removeEventListener` on `global` |
| `utils/devices.ts` top-level UA sniff | safe — `navigator.userAgent`/`vendor` undefined in RN → `vendor` falsy → IIFE returns early | none |
| `settings.ts` environment ternary (`document.head` etc.) | safe — guarded by `DomDocumentHelper.isAvailable()` | none (and: **never define `global.document`** — its absence routes core into SSR-safe paths) |

The shim is **listener no-ops only**. Peer review killed the earlier ResizeObserver stub: `question.ts#initResponsiveness` is document-guarded (returns in RN), `responsivity-manager.ts:19` is typeof-guarded, and the unguarded constructions (`scroll.ts#setRootElement`, `survey.ts#afterRenderSurvey`) sit on DOM-render entry points our native renderer must never call. A global no-op observer would also make core feature-detection succeed while callbacks never fire — silently inert responsivity is worse than absence. If a native call graph ever needs resize signaling, that's an explicit native resize adapter task, not a global stub.

Shim principles: **patch only what's missing** (never override an existing global), **idempotent**, **no `document`**, **zero imports** in the shim module.

## Module layout

```
src/core/
  shim.ts     — side-effect module: applySurveyCoreShims() + auto-invoke on import. Imports nothing.
  facade.ts   — `import './shim';` then `export * from 'survey-core';`
src/index.tsx — re-exports public API from facade (never from 'survey-core' directly)
```

- `shim.ts` exports `applySurveyCoreShims(): void` (A2 way 3) and calls it at module scope (ways 1+2 ride on this).
- `facade.ts`: ECMAScript evaluates module requests in **source-occurrence order** (including `export … from` requests), so the `import './shim'` request (first occurrence) evaluates before `export * from 'survey-core'` pulls core in — for this acyclic graph. Babel CJS / RN-preset output emits `require('./shim')` before `require('survey-core')`, preserving order. The packaged-entry tests execute the emitted artifacts as the regression proof; the spec claim alone is not relied on.
- Internal rule: **all** survey-core touches go through `./core/facade`. ESLint: `no-restricted-imports` with patterns `survey-core`, `survey-core/*` (covers `import type` too) PLUS `no-restricted-syntax` selectors for `require('survey-core…')` and dynamic `import('survey-core…')`, scoped to `src/**` with `src/core/facade.ts` excepted. Each escape hatch (static, type, require, require.resolve, dynamic import, subpath) gets a lint fixture test.

## package.json wiring

Literal exports addition (custom source condition key matches the existing root entry):

```json
"./shim": {
  "iotashan-llc-react-native-survey-library-source": "./src/core/shim.ts",
  "types": "./lib/typescript/src/core/shim.d.ts",
  "default": "./lib/module/core/shim.js"
}
```

- **ESM-only package** (builder-bob `module` target, `esm: true`, emits `lib/module/package.json` with `type: "module"`; no `commonjs` target, no `require` condition). "CJS" in A2 is hereby narrowed to *Metro/Babel-transformed source semantics* (how every RN consumer + jest actually loads us) — there is no Node-CJS entry to support, and none is added speculatively. Plan doc A2 wording updated to match.
- `sideEffects`: `["./src/core/shim.ts", "./lib/module/core/shim.js", "./src/core/facade.ts", "./lib/module/core/facade.js"]` — the facade is itself side-effectful (its evaluation applies the shim); allowlisting only the leaf would let an optimizer drop the chain from a re-exporting barrel. Note: Metro (the only bundler in scope, iOS+Android-only lib) does no `sideEffects` pruning today — this field is correctness-in-principle for any future tree-shaking consumer; no bundler CI test is added for a bundler consumers don't use. If a tree-shaking bundler enters scope, add a bundle-and-execute test then.
- `peerDependencies`: `"survey-core": ">=2.5.32 <2.6.0"`; `devDependencies`: `survey-core@2.5.33` exact. **Held against peer objection through both rounds** (reviewer wanted a per-patch cap; final objection: "CI detects after exposure, not an admission gate" — factually correct). Coordinator decision: a per-patch cap forces a release of ours per upstream patch and blocks consumer upgrades until then, forever; the exposure window only hits consumers who bump survey-core to a brand-new patch before the `latest-2.5.x` CI job flags it, and the RED-baseline test makes any new require-time DOM access fail loudly there. Accepted risk, mitigations: CI matrix pins [2.5.32, 2.5.33] + `latest-2.5.x` job; README advises consumers to pin survey-core in their lockfile; band-expansion gate for minors.

## Import-order contract (consumer-facing)

1. **Renderer-first (default)**: import anything from `@iotashan-llc/react-native-survey-library` before touching survey-core → shim auto-applied.
2. **Model-first**: consumer imports `survey-core` first (e.g. builds `SurveyModel` in a store module) → crash at survey-core require. Contract: `import '@iotashan-llc/react-native-survey-library/shim';` above the first survey-core import. Documented in README + DIFFERENCES.md.
3. Odd loaders / conditional require: call `applySurveyCoreShims()` manually before first survey-core touch.

## Test plan (TDD; tests written first)

Fixture fidelity facts (verified, Node 22): Node's `globalThis` has **no** `addEventListener` and is **not** an EventTarget; `window` undefined; `navigator` exists but Node-shaped. RN 0.86 has `window === global`, `navigator.product === 'ReactNative'`, rAF, no `document`, no `window.addEventListener`. The fixture **constructs** the RN shape from explicit descriptors rather than assuming either baseline:

`test-utils/rn-globals.ts` — `withRnShapedGlobals(fn)`: saves originals; sets `global.window = global`, `navigator` → `{ product: 'ReactNative', maxTouchPoints: undefined }`-shaped, supplies `requestAnimationFrame` if absent; deletes `addEventListener`/`removeEventListener`/`ResizeObserver` if present; **asserts** the resulting preconditions (`typeof document === 'undefined'`, `window === global`, no `window.addEventListener`, globalThis not an EventTarget instance); runs `fn` under `jest.isolateModules`; restores descriptors in `finally`.

**Unit suite** (jest node env, installed survey-core, babel-transformed src):

1. RED baseline: RN-shaped globals + bare `require('survey-core')` → throws (locks in why the shim exists; alerts if upstream fixes RN detection).
2. Shim-first: `require('./src/core/shim')` then `require('survey-core')` → loads; `new Model(json)` headless smoke (page, setValue, complete) + invalid-required-submit asserts the KNOWN `settings.environment` failure **specifically** (TypeError whose message names the `environment`/`root` destructure — not merely "throws"), so the tripwire can't be satisfied by an unrelated error (documents the 1.2 dependency, keeps ledger honest).
3. Facade alone → loads (order inside facade works under babel-CJS semantics — this IS the Metro/Babel-consumer proof for transformed source).
4. Idempotence + non-override: shim twice → no throw; pre-existing `global.addEventListener` sentinel NOT replaced.
5. No document after shim.
6. Lint fixtures: every survey-core escape hatch outside facade fails lint (static, type-only, subpath, require, require.resolve, dynamic import); facade passes.

**Packaged suite** (`test:packaged`, CI after build — the exports-map + ESM proof):

7. `yarn pack` → install tarball + survey-core into a temp consumer dir → spawn `node --input-type=module` child processes whose preamble builds the RN shape (same descriptor logic, inline), then by **package specifier** (never lib paths — the exports map itself is under test):
   - (a) `import('@iotashan-llc/react-native-survey-library/shim')` → `import('survey-core')` → exit 0 (model-first contract).
   - (b) `import('@iotashan-llc/react-native-survey-library')` → `import('survey-core')` → exit 0 (renderer-first; proves the packed facade chain applies the shim under real ESM. NOT a tree-shaking/sideEffects proof — Node `import()` consults no bundler config; see wiring note).
   - (c) bare `import('survey-core')` → nonzero exit (negative control).
8. Real-device/Metro fidelity is NOT claimed by 7 — Node can't model Hermes. Covered by: 0.2 release-gate (packed tarball → Expo app → native launch smoke) and example app (1.17). Noted here so nobody mistakes case 7 for a Metro test.

## Non-goals / deferred

- No `document` stub, ever.
- No ResizeObserver/MutationObserver/IntersectionObserver stubs (see ledger).
- No viewport shims — renderer supplies layout natively (M1+).
- No Node-CJS package entry.
- No patching of survey-core (A1: unmodified peer).

## Amendment (task 1.2, A15): `settings.environment` stub via the facade

`applySurveyCoreShims` now takes an OPTIONAL `settings` argument. When passed survey-core's `settings` singleton it applies `settings.environment ??= { root/rootElement/popupMountContainer/svgMountContainer/stylesSheetsMountContainer: undefined }` — fields undefined (downstream readers are optional-chained), object defined (the `survey.ts:5872` destructure survives). The FACADE passes `settings` after `export * from 'survey-core'` evaluates; `shim.ts` itself still has ZERO imports and the self-invocation stays global-only, so the `/shim` subpath remains zero-core-import (its tripwire test now locks that boundary: shim-only invalid-submit still throws; the facade path does not — see `src/core/__tests__/environment-stub.test.ts`). The ledger row "`survey.ts:5872` … throws" is resolved by this amendment plus the 1.2 bridge (which cancels the funnel before the destructure is even reached). `document` is still never defined. Full design: `docs/design/1.2-lifecycle-bridge.md`, piece 3.
