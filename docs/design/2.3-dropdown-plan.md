   `showInputFieldComponent` and `showSelectedItemLocText` branches,
   render `dropdownListModel.inputStringRendered` when non-empty, THEN

   control). Regression test: preselected value renders its text
   through each branch.
B. LAZY-LOAD DEDUPE MECHANISM (codex r2 #2 — the v2 "release on
   onChoicesLazyLoad resolution" seam does not exist; the observer
   returns void and core fires the event synchronously): core marks
   in-flight through `setIsChoicesLoading(true/false)` around each
   `_loadQuestionChoices` (dropdownListModel.ts:73,86) — but the flag
   is PRIVATE (`_isChoicesLoading`; no public typing — round 3). The
   public proxy is `question.isReady`, which core toggles false/true
   around lazy loading. RN gate in ListPicker's end-reached handler:
   skip the observer call while `(model.locOwner as Question)?.isReady
   === false` (locOwner IS the owning question —
   dropdownListModel.ts:336). Generic (tagbox inherits), public API
   only. Core's own skip-window check (updateQuestionChoices:112-113)
   remains the paging guard. Pinned by a watchlist entry for
   `Question.isReady` + a delayed-`setItems` dedupe test: repeated
   endReached yields only skip 0; after resolution the next call
   yields skip 25.

## Round-1 corrections (codex 8 + gemini 2, reconciled)

1. TOUCH ORACLE FIXED: `needRenderInput` is TRUE for editable dropdowns
   in 2.5.33 — IsTouch only sets `inputMode='none'` (non-editable
   input) and popup search arrives via `setSearchEnabled` on OPEN
   (dropdownListModel.ts:255-258). RN control therefore renders VALUE
   TEXT (selectedItemLocText / inputStringRendered display) or
   placeholder; hint prefix/suffix + editable inline filter are dropped
   (DIFFERENCES — typing happens in the overlay's search box).
2. onClear event: core dereferences ONLY preventDefault()/
   stopPropagation() — synthetic `{ preventDefault: noop,
   stopPropagation: noop }`; clear gate mirrors core:
   `allowClear && !isEmpty() && !isInputReadOnly`.
3. LAZY-LOAD OWNERSHIP FIXED: DropdownListModel itself assigns
   `listModel.loadingIndicatorVisibilityObserver` (its update-choices
   loader) — ListPicker's endReached call already drives core's load
   path. The DEDUPE mechanism is defined in Round-2 correction B
   (public `question.isReady` gate — supersedes any earlier wording).
4. REACTIVITY FIXED: `getStateElements() = [question,
   dropdownListModel]` (question-level props — allowClear, readOnly,
   placeholder locs — change on the QUESTION; the native base does not
   auto-subscribe it).
5. OPENER FOCUS SEAM: extend `registerPopup(popup, stack, options?)`
   with `{ openerHandle?: () => number | null }`; the bridge stores it
   on the payload; OverlayHost restores a11y focus to the opener on
   final removal. (Small additive 2.1 extension + its own tests.)
6. VALUE RENDERING: honor `question.showInputFieldComponent` →
   `RNElementFactory.createElement(question.inputFieldComponentName,
   { item: dropdownListModel.getSelectedAction(), question })` when
   registered (fallback diagnostic + locText); else
   `showSelectedItemLocText` → renderLocString(selectedItemLocText);
   else placeholder. OTHER comment area via the established
   comment-item adapter (checkbox/radiogroup pattern) + test.
7. EDITOR BUTTONS: render core's chevron/clear as model-driven
   affordances (enabled/visible from the model; localized
   accessibility labels), not hand-rolled statics.
