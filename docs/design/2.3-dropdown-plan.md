Status: PLAN v1 (pair review pending). Not CORE — standard task plan,

## Upstream oracle

`survey-react-ui/src/dropdown-base.tsx` + `reactquestion_dropdown.tsx`;
model `survey-core/src/dropdownListModel.ts` (created lazily by
`QuestionDropdownModel.dropdownListModel`). Under `_setIsTouch(true)`
(facade): `displayMode='overlay'`, `isFocusedContent=true`, search
lives INSIDE the popup (`setSearchEnabled` on open — the CONTROL's
filter input is not rendered: `needRenderInput` stays false on touch;
verify against 2.5.33 and pin).

## Component: `DropdownQuestion` (template route, dispatchKey 'dropdown')

- `getStateElement() = question.dropdownListModel` (web parity — the
  VM, not the question).
- Control (replaces web's div+input): a `Pressable`
  (testID `sv-dropdown-control`) rendering:
  - value: `question.showSelectedItemLocText` →
    `renderLocString(selectedItemLocText)`; else placeholder
    (`dropdownListModel.placeholderRendered`) in placeholder styling.
  - chevron marker + CLEAR button (`allowClear` gate —
    `question.allowClear && !question.isEmpty()`): clear presses
    `dropdownListModel.onClear(event)` with a synthetic event object
    ({ keyCode: 0, preventDefault: noop, stopPropagation: noop } —
    core's onClear dereferences these; verify exact fields).
  - press → `dropdownListModel.onClick()` (toggleVisibility — popup
    show routes through the question-scoped bridge below).
- READONLY: `question.isInputReadOnly` → no press handler,
  `readOnlyText` rendering (web renderReadOnlyElement parity).
- a11y: `ariaQuestionRole/Label/Expanded/Required/Invalid` translated
  via the established ariaToBool pattern; control
  `accessibilityRole="button"` + `accessibilityState.expanded`.
- POPUP BRIDGE (question scope): on mount register
  `dropdownListModel.popupModel` into the OverlayContext stack
  (`registerPopup`); unregister on unmount (semantic close). Content
  'sv-list' dispatches the 2.1 ListPicker automatically. The component
  wraps in a `DropdownQuestionElement`-style context consumer like
  ListPickerElement (class component + contextType conflict).
- OPENER FOCUS RESTORATION (2.1 D8 seam): pass the control ref;
  on final dismissal restore a11y focus to the control
  (AccessibilityInfo.setAccessibilityFocus).
- LAZY-LOAD owner adapter: `choicesLazyLoadEnabled` — ListPicker
  already fires `loadingIndicatorVisibilityObserver(true)`; the
  DropdownListModel installs its own observer via
  `initLoadingIndicatorVisibilityObserver` ONLY behind
  IntersectionObserver (absent in RN — action.ts:549). Verify which
  seam core uses for `updateItemsOnScroll`/load-more in 2.5.33 and
  wire the RN equivalent: expected — call
  `dropdownListModel.loadQuestionChoices`-family through the observer
  the picker triggers; DEDUPE in-flight pages (design note from 2.1).
- MASKED/OTHER: `question.otherItem` flows through core choices +
  comment area (`renderOther` parity — reuse existing comment-item
  pattern from checkbox/radiogroup if present).
