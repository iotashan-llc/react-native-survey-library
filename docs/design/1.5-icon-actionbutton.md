# Task 1.5 design — RNIcon + ActionButton primitives (A15, icons half)

Status: **revised after codex security review** (sol@high, `revise` — 3 major + 3 minor, all addressed; see "Review round 1 amendments" at the end). Consumes 0.4 (`SurveyElementBase`), 0.7 (button recipe via provider/`selectStyles`), 0.9 (trust-tier framing, diagnostics seam). Owner: task agent.

## Verified upstream facts (v2.5.33 checkout + installed package)

- **Icon registry**: `svgbundle.ts` exports `SvgRegistry: SvgIconRegistry` and `SvgThemeSets: {[set]: {[id]: rawSvg}}`. `SvgIconRegistry.icons` stores **`<symbol id="icon-x" …>`-wrapped** strings (`registerIconFromSvg` rewraps the consumer's `<svg …>…</svg>`); `SvgRegistry.registerIcon(id, svg, setVersion="v2")` ALSO stores the **raw** `<svg>` string in `SvgThemeSets[setVersion][id]` via `addIconsToThemeSet`. Registration fires `SvgRegistry.onIconsChanged` (`EventBase`) only via `registerIcons()`' `updateMarkup()`; `registerIcon()` (singular) does NOT fire it — resolution must not assume an event per registration.
- **Bundled sets**: `survey-core/icons/iconsV2` (and `iconsV1`) subpath exports `icons: {[name]: rawSvgString}` — 61 icons each, svgo-processed at upstream build, keys are canonical size-suffixed names (`chevrondown-24x24`, `smiley-rate1-24x24`, …). Verified importable in this repo (ESM `fesm/icons/*.mjs` + CJS `icons/*.js` both in the exports map). The web 2.x renderer registers **V2 only** (`survey-react-ui/src/reactSurvey.tsx:20 SvgRegistry.registerIcons(iconsV2)`).
- **Name resolution**: `utils/icons.ts` exports `getIconNameFromProxy(iconName)` (public on the package root): `settings.customIcons` indirection first (`getCustomNewIconNameIfExists` — checks the raw name, then the renamed name), then `getNewIconName` (strips `icon-` prefix, maps ~140 legacy names through `renamedIcons` — this is where size-suffixed canonical names come from, e.g. `icon-clear_16x16` → `icon-clear-16x16`, `icon-chevron` → `icon-chevrondown-24x24`). Returns a **prefixed** `icon-<canonical>` name; registry keys are UNprefixed.
- **`settings.customIcons` is a name→name remap** (`settings.customIcons["icon-redo"] = "custom-redo-icon"`), NOT an SVG payload. The custom SVG itself arrives via `SvgRegistry.registerIcon(...)` — that is the consumer-content surface.
- **Action model** (`actions/action.ts`): `Action extends BaseAction extends Base` — `@property`-backed `iconName`, `iconSize` (default 24), `title`/`locTitle`, `tooltip`, `enabled`/`disabled` (`disabled === enabled !== undefined && !enabled`), `visible`, `isVisible` (`visible && mode !== 'popup' && mode !== 'removed'`), `active`, `pressed`, `ariaChecked/ariaExpanded/ariaRole` (default `"button"`), `disableTabStop`, `hasTitle` (model-owned show-title logic), `getTooltip() = tooltip || title`. `doAction(args)` unwraps `args.originalEvent ?? args`, calls `action(this, evt.isTrusted)`, then `evt.preventDefault(); evt.stopPropagation()`. `doMouseDown`/`doFocus` track mouse-vs-keyboard focus origin.
- **Web ActionBar item** (`survey-react-ui/action-bar-item.tsx`): `SurveyActionBarItem extends SurveyElementBase`, `getStateElement() → item` (the Action), button renders SvgIcon + locTitle viewer; `SurveyAction` wrapper owns root css (`sv-action--hidden`) + the DOM `updateModeCallback` shrink machinery.
- **Web SvgIcon**: renders `<svg><use xlink:href="#icon-x"></use></svg>` against the DOM sprite; `createSvg` resolves via `getIconNameFromProxy` and sizes via `size || width || 16`. Pure DOM — nothing portable except the resolution call.

## RNIcon

### Icon source strategy

Bundled core icons come from the **`survey-core/icons/iconsV2` subpath** (raw SVG strings — exactly what `SvgXml` needs), imported through a new **icons-facade `src/core/icons.ts`** (same pattern/ESLint exception as `src/core/themes.ts`: `import './shim'` first, only file allowed to touch the subpath). V2 only, mirroring the web 2.x renderer; consumers wanting V1 can `SvgRegistry.registerIcons(iconsV1Icons)` themselves and resolution picks them up via the registry paths below.

Resolution (`resolveIconSvg(iconName)` — pure, unit-testable, exported for tests):

1. `getIconNameFromProxy(iconName)` (facade) → canonical `icon-<key>`; strip prefix → `key`. This is the ONLY name-mapping logic — `settings.customIcons` + `renamedIcons` + size-suffixed canonical names all ride core's own resolution, never duplicated here (invariant 6 spirit).
2. Raw-source lookup, first hit wins — **`SvgRegistry.icons` is the AUTHORITATIVE consumer source** (review amendment: every upstream registration path funnels its processed icon there, so it alone carries global last-write-wins chronology; `SvgThemeSets` is a side store only `registerIcon` touches and a stale set entry must never shadow a later registry write):
   1. `SvgRegistry.icons[key]` — `<symbol …>` string; unwrapped back to `<svg …>` by tag rewrite (`registerIconFromSvg` produced it from an `<svg>` by exactly that transform, so the reverse is lossless);
   2. `SvgThemeSets['v2'][key]`, then `SvgThemeSets['v1'][key]` (residual raw source for hosts that populated the sets directly); tried under canonical + `icon-`-prefixed key;
   3. bundled V2 map.
3. Miss → `null`; component renders **null** + one-shot structured diagnostic `unknown-icon` (deduped per resolved key, `reportDiagnostic` seam) — never throws (invariant 9 spirit).

Registry liveness: RNIcon subscribes `SvgRegistry.onIconsChanged` (commit-phase add/remove, 0.4 discipline — plain `React.Component`, NOT `SurveyElementBase`: the registry isn't a `Base` and needs no property-callback machinery) and re-resolves on fire. Because `registerIcon` (singular) doesn't fire the event, docs tell hosts to register icons before mounting (web has the same staleness: the sprite renders from a mounted component).

### Sanitization decision (A15 ⨯ 0.9 sink matrix)

Two trust tiers, per 0.9's content-origin framing:

| Source | Origin | Policy |
|---|---|---|
| Bundled `survey-core/icons/iconsV2` map | trusted-library-generated (same tier as core code; svgo-built) | **passthrough** — byte-identical into `SvgXml`, no parse |
| `SvgThemeSets` / `SvgRegistry.icons` (anything consumer-registered) | HOST-code input (`registerIcon` calls) — survey-JSON authors CANNOT reach this sink (JSON has no icon-SVG field; `customIcons` remaps names only) | **parse-validate, svg-only allowlist** before `SvgXml` |

Decision: **validate, don't trust-declare.** Rationale: (a) `SvgXml` is a real XML-parse + native-render sink, and react-native-svg supports `<Image href>`/`<Use href>` — an icon string with an external href would perform network fetches OUTSIDE the A11 URI policy (auto-fetch sink smuggled through a render sink); (b) "trusted consumer input" in practice means "pasted from an icon pack" — a cheap structural pass closes real risk without burdening hosts; (c) the parser (`htmlparser2`, xmlMode) is already a production dependency (0.9), icons are tiny, and results are cached per raw string. This is defense-in-depth on top of host trust, mirroring `SanitizedHtml`'s "immutable safety pass always runs" posture, not a reclassification of the host as hostile.

Policy (`sanitizeIconSvg(raw)` in `src/security/sanitize-svg.ts`, exported result `{ xml: string | null, diagnostics }`):

- Parse `htmlparser2` xmlMode (single parse; no entity/namespace surprises — same family as `sanitize-html.ts`).
- Root must be a single `<svg>` element → else invalid (`icon-svg-invalid` diagnostic, resolution miss).
- **Element allowlist** (dropped subtree + `icon-svg-sanitized` diagnostic otherwise): `svg g path rect circle ellipse line polyline polygon defs linearGradient radialGradient stop clipPath mask pattern symbol use title desc text tspan`. Notably excluded: `image` (remote fetch), `foreignObject` (arbitrary embed), `script`, `animate*` family, anything unknown.
- **Attribute policy**: drop `on*` (event handlers), drop `href`/`xlink:href` unless a local fragment (`#id` — keeps `<use>` of local `<defs>`); everything else passes through (`d`, `fill`, `stroke`, `viewBox`, `transform`, `style`, … — react-native-svg ignores what it doesn't support; CSS `var()` inside `style` won't resolve on RN, which is a fidelity note, not a security issue — core's own `timercircle` has this and it's on the TRUSTED path anyway).
- **Resource bounds, fail-closed** (review amendment — mirrors 0.9's posture): `SVG_RESOURCE_BOUNDS` = source ≤ 64KB (pre-parse), ≤ 512 elements, depth ≤ 32 (doubles as the recursion-safety bound for both the filter walk and dom-serializer), ≤ 64 attrs/element, attr value ≤ 32KB. Any exceeded bound rejects the WHOLE input (`icon-svg-invalid`), never a partial tree. `sanitizeIconSvg` NEVER throws: bound sentinels and any parse/filter/serialize failure are caught and returned as `icon-svg-invalid` (RNIcon's never-throw contract).
- **`style` attribute validated, not passed blind** (review amendment): react-native-svg's own parser throws on a declaration without `:`; every `;`-separated declaration must be `property: value` or the attribute is dropped whole with a diagnostic.
- **Exact local-fragment grammar** for kept `href`s: `/^#[A-Za-z_][A-Za-z0-9_.:-]*$/` — not merely "starts with `#`" (bare `#`, embedded whitespace rejected).
- Serialize with `dom-serializer` (xmlMode; already in the htmlparser2 dependency tree, now declared explicitly as a direct dependency).
- Module-scope cache `Map<raw, result>` (icons are small and few; hard cap with reset guards pathological registries).

### Component surface

```ts
interface RNIconProps {
  iconName: string;
  size?: number | string;      // default 24 (BaseAction.iconSize default; web createSvg's bare fallback is 16 — divergence documented)
  fill?: ColorValue;           // optional monochrome override; theme wiring stays per-component (A7)
  title?: string;              // a11y label; absent → decorative (accessible={false}), mirroring web role="presentation"
  style?: StyleProp<ViewStyle>;
  testID?: string;
}
```

`react-native-svg` is **lazy-required inside RNIcon at first render** (module evaluation deferred; Metro still statically resolves the literal `require` — same narrowed claim as `SanitizedHtml`), and RNIcon.tsx is the ONLY file allowed to import it (new ESLint boundary, same selector family as the facade/native-html rules).

## ActionButton

`ActionButton extends SurveyElementBase<ActionButtonProps>` (0.4 base; mirrors web `SurveyActionBarItem`):

- `getStateElement() → props.action` — `iconName/title/enabled/visible/pressed/active/mode` property changes re-render through the ported callback mechanism; unmount unsubscribes (0.4 owns the discipline).
- `canRender()`: requires `action`; renders **null when `!action.isVisible`** (RN has no css-hidden wrapper; the model getter — `visible && mode ∉ {popup, removed}` — is consumed, never re-derived).
- **Pressable + 0.7 button recipe**: render-time `selectButtonStyles(recipes.button, { pressed: nativePressed || action.pressed, focused: nativeFocused, disabled: action.disabled, small, variant }, mode)` from the inherited `themeContext` (companion-amendment contextType). `variant`/`small` are **usage-site props** (`'default' | 'action' | 'danger'`, default `'default'`) per the 0.7 fixture's model; `pressed`/`focused` are native interaction state (A7) — `action.pressed` (model-derived, e.g. dropdown-open) ORs into the pressed input.
- **Style split**: the recipe's composed array mixes container (padding/background/radius/shadow) and text (color/font*) styles because web buttons inherit text styles; RN doesn't. `partitionButtonStyles(flattened)` (exported, unit-tested) splits by a text-style key allowlist — container object → Pressable `style`, text object → title `Text`/icon fill default. A12 slots compose LAST per side: new `actionButton: { container?, icon?, title? }` in `SurveyComponentStyles` (existing `button.button` slot stays for plain-button ports).
- **Content**: `RNIcon` when `action.iconName` (size `action.iconSize`, no title — the button carries the label) + title `Text` when `action.hasTitle` (model-owned logic), rendered through the inherited `renderLocString(action.locTitle, …)` seam — plain text now, upgraded transparently by 1.6.
- **Events**: `onPress → action.doAction(nativeActionEvent())` where `nativeActionEvent()` supplies `{ originalEvent: { isTrusted: true, preventDefault: noop, stopPropagation: noop } }` — native presses are genuinely user-initiated (trusted), and RN needs no default-prevention; the shim satisfies `doAction`'s DOM-shaped contract without touching the RN synthetic event. `onPressIn → action.doMouseDown({})`, `onFocus → doFocus + focused=true`, `onBlur → focused=false` (keeps core's mouse-vs-keyboard focus-origin bookkeeping intact).
- **A11y**: `accessibilityRole` mapped from `ariaRole` (`button`/`checkbox`/`radio`/`tab`/`link`/`switch`/`menuitem` known, default `button`), `accessibilityState { disabled, checked: ariaChecked, expanded: ariaExpanded, selected: active }`, `accessibilityLabel = action.getTooltip()` (covers icon-only), `accessibilityLabelledBy = action.ariaLabelledBy` (Android nativeID relationship; iOS ignores), `disabled={action.disabled}` on Pressable, `focusable = !action.disableTabStop`.
- **`active` vs `pressed`** (review amendment): `action.active` is the model's SEMANTIC selection/active flag (upstream itemActive) → `accessibilityState.selected`; `action.pressed` is a transient pressed/dropdown-open VISUAL (upstream itemPressed) → ORed into the recipe's pressed input only, never surfaced as selection. Visual active styling awaits an `active` fragment in the 0.7 button-recipe fixture (owned by the ActionBar item task) — inventing a mapping onto the pressed fragment would misrepresent the fixture's locked legal states.

### Non-goals (owned elsewhere)

ActionBar container + adaptive shrink (`updateModeCallback`/`updateDimensions`/`minDimension` — DOM-measurement machinery; the RN ActionBar task designs its `onLayout` analog); popup/dropdown actions (`popupModel`, `component` dispatch — A9 overlay, M2); `markerIconName`/sub-items; rich locTitle (1.6); per-usage variant selection (nav/matrix/panel tasks pass `variant`).

- ESLint: `react-native-svg` restricted to `src/components/RNIcon.tsx` (+ its fixtures if needed) — full escape-hatch selector family; lint-fixtures test extended.
- Jest: `react-native-svg` added to `transformIgnorePatterns` exception; component tests run against a root manual mock (`__mocks__/react-native-svg.tsx` — `SvgXml` as a props-capturing stub) so assertions target our contract (resolved xml/size/fill/a11y) instead of host-component internals; the real lib is exercised by the A14 packaged/example gates.
