## Sanitizer (A11) — `src/security/sanitize-html.ts`

**Single-parse AST pipeline (round-1 rework — no parse→serialize→reparse):** parse ONCE with the exact TRE/htmlparser2 configuration the renderer uses, filter into a PRIVATE reconstructed allowlisted AST (positive reconstruction — allowlisted nodes/attrs are COPIED into new nodes; everything else dropped, active subtrees dropped whole, never unwrapped), pass that AST to the renderer's DOM-source input. No serializer divergence, no post-validation mutation window. `htmlparser2`/`domhandler` become DECLARED dependencies (currently transitive-only — undeclared import was a defect; hoisting hid it).

- Tag allowlist as before (`p div span br hr h1-h6 ul ol li blockquote pre code strong b em i u s sub sup table thead tbody tr td th a img`). Always-dropped-as-subtrees now explicitly includes foreign/raw-text/container elements: `svg math foreignObject template noscript meta link base style script iframe object embed form input textarea select button`, comments, doctypes, CDATA, processing instructions.
- Per-tag attribute allowlist; duplicate attributes: first occurrence wins, rest dropped + diagnostic; `width/height/rowspan/colspan` validated as bounded integers (1–10000 / spans 1–1000); `style` stripped unconditionally (fidelity loss documented; future CSS support = separate allowlisted property/value parser, no `url()`, no imports — recorded, not built).
- **Resource bounds (attacker-controlled input)**: pre-parse source cap (256KB default, configurable down only), node count (5000), depth (32, enforced during parse or iterative traversal), attributes per element (16), attr value length (2048), decoded text total (512KB), images per document (32), table dimensions (200×64). Exceeding any → render as plain text extraction + diagnostic (never partial-render).
- Bypass corpus (round-1 additions): htmlparser2 implied-close/table-scope tricks, foreignObject/SVG/MathML smuggling, `template` content, entity double-decode, attr-value breakout, tab/newline/NUL-embedded schemes, uppercase/mixed schemes, protocol-relative, data: with whitespace/percent-escapes/extra params.
- **NO raw opt-out** (round-1 critical): the old `dangerouslyDisableHtmlSanitization` is gone. Trusted-content relaxation = a `relaxedFormatting` policy that may widen the FORMATTING allowlist (more tags/attrs) but an **immutable safety pass always runs**: active elements/handlers removed, URI policy enforced, renderer JS/inline-CSS off. The `javascript:` rule is now genuinely unoverridable because no path skips the safety pass.

## Central URI policy (A11) — `src/security/uri-policy.ts`

**Scheme AND origin policy** (round-1 critical: A11 says scheme/origin; schemes alone let author HTML trigger silent network fetches to arbitrary hosts):

- `validateUri(raw, context, config)` returns `{ ok: true, canonical, scheme, origin } | { ok: false, reason }` — **sinks MUST consume `canonical`** (validate-one/use-another bug closed; adapter/tests enforce). Rejection (not stripping) of embedded C0/NUL/BOM/Unicode-separator chars; lowercase-normalized scheme; parsed origin.
- **Per-context defaults**: `link` (event-only, human-mediated) = broad scheme set (`https http mailto tel`), no origin restriction. **Automatic-fetch contexts** (`image`, `background`, `choicesByUrl`, `video`) = `https:` only by default + **origin allowlist** (default: empty = same-config-declared origins only; consumers list allowed origins at `<Survey>` config). Denied by default in fetch contexts: `http:`, IP-literal hosts, `localhost`/`.local`/RFC-1918/link-local ranges, URL-embedded credentials (`user:pass@`), non-default ports unless origin-listed.
- **Immutable per-context deny set**: `javascript: vbscript: file: about: blob: filesystem: intent: content: jar: data:` (data: excepted ONLY for `image` under the strict data-image rule below) — not overridable by any config.
- **Data-image rule** (round-1 tightening): `data:image/(png|jpe?g|gif|webp);base64,<payload>` — strict base64 alphabet + padding, no whitespace/percent-escapes/extra parameters; cap on DECODED bytes (1MB, computed from encoded length pre-decode); magic-bytes check matches declared MIME before handing to the native decoder.
- `choicesByUrl` (round-1 correction — pre-model JSON validation is INSUFFICIENT: core substitutes `{variables}` at REQUEST time, choicesRestful.ts:205-228): policy enforces at BOTH ends — (a) JSON-time lint: substitutions forbidden in scheme/authority/port positions (path/query only, encoded); (b) request-time validation of the FULLY RESOLVED URL. **Round-2 correction on mechanism**: `settings.web.onBeforeRequestChoices` receives an already-created XHR, returns void, and has NO documented cancellation contract — the M1 wiring task must EMPIRICALLY prove abort-before-send with a zero-network-side-effect test (mock transport asserting no open/send); if the hook can't guarantee it, use the ChoicesRestful ajax-seam replacement (documented core callback, not a patch). `settings.web` is GLOBAL: our installation composes with any pre-existing host hook (chain, never clobber) and routes per-survey policy via a registry keyed by the requesting survey.
- **Redirects fail CLOSED** (round-2): an allowed origin redirecting to a denied one is a policy breach, not residual risk. Automatic-fetch contexts use `redirect: 'manual'`/error semantics where the transport supports it; where per-hop validation is impossible on a platform, that fetch context fails closed (fetch refused + diagnostic) rather than following. Applies to images/video/background, not just choicesByUrl.
- **Relative URLs** (round-2): RN has no document origin. Automatic-fetch sinks require ABSOLUTE URLs by default; a consumer-supplied trusted `baseUrl` (config, itself policy-validated at set time) enables relative resolution — resolve once → canonicalize → validate the absolute result. Without `baseUrl`, relative fetch URLs are rejected with a diagnostic naming the config. Event-only links pass relative values through to the host unresolved (host owns interpretation).
- No auto-navigation anywhere; `navigateToUrl` family surfaces as events (M1 wiring).
