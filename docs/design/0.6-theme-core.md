## Default-table contract (round-1 critical 1 — resolved)

**Chosen oracle: CSS-cascade parity.** `defaults.ts` vendors the SCSS fallback-chain table: for every `--sjs-*` variable consumed by default-theme SCSS, the value the cascade would produce with NO theme applied (terminal literals; where a default is an expression over other vars, it stays an EXPRESSION — see registry below). `resolveTheme(undefined)` = this table resolved — the web "no theme" look. It is ≈DefaultLight but NOT byte-equal (documented; secondary alphas differ). Sparse custom themes therefore fall back exactly as the web cascade does. DefaultLight-equality is NOT claimed anywhere (round-0's inconsistency removed); a golden pins `resolveTheme(undefined)` on its own terms.

## Variable registry (round-1 critical 2 — classified, not one-literal-per-key)

`registry.ts` classifies every known variable:

| class | examples | default representation |
|---|---|---|
| `preset-base` (the 82) | `--sjs-primary-backcolor` | terminal literal |
| `semantic-derived` (SCSS vars defined as expressions over others) | `--sjs-font-size`-derived article fontSizes, editor/panel backcolors | **expression, evaluated after sparse-overlay** (so overriding a base key flows into derived defaults, matching the cascade) |
| `context-dependent` | header title/description colors | NOT resolved to one value in 0.6 — exposed as raw + helper (0.7 recipes evaluate per context) |
| `web-only` (motion/runtime keys) | animation/transition vars | ignored, documented list |
| legacy aliases | `--primary`, `--base-unit`, `--font-family` | in the dereference graph as alias edges |

## Module layout (pure TS, zero `react-native` imports, ESLint-enforced)

```
src/theme-core/
  registry.ts   — variable classification + expected-format grammar per key + source refs (scss file:line per entry, per A7's source-documented rule)
  defaults.ts   — the vendored cascade table (literals + expressions)
  parse.ts      — bounded full-match grammars (below)
  resolve.ts    — resolveTheme(theme?) → ResolvedTheme (pure DATA — no functions in output)
  helpers.ts    — pure standalone fns: spacing(baseUnit, multiplier); evaluateVarExpression(rawVariables, expr) for 0.7's context-dependent lookups
  themes-facade → src/core/themes.ts: `import './shim'; export * from 'survey-core/themes';` — the ONLY module allowed to import the survey-core/themes subpath (ESLint exception added alongside the facade rule); re-exports a named THEME_MANIFEST of exactly 40 names (internal hash sentinel excluded; count asserted)
```

Theme-core imports ITheme as `import type` from the facade — loads without survey-core evaluated (tested).

## Resolution algorithm

1. `rawVariables` = the COMPLETE unevaluated cascade environment: every registry entry's raw definition (literals AND unevaluated expressions for semantic-derived/context-dependent entries) overlaid with `theme.cssVariables`. Returned as-is so `evaluateVarExpression(rawVariables, expr)` reproduces cascade behavior for ANY variable, incl. derived/context defaults (round-2 fix: a literals-only merge could not).
2. **var() dereference** (round-1 rework): tri-state DFS over the full variable graph (white/grey/black), bounded by graph size — no arbitrary hop cap. Cycle members marked invalid FIRST, then the referring expression's own fallback applies; no fallback → registry default → diagnostic. `var(--x, fallback)` fallback split at the first TOP-LEVEL comma (paren-aware); nested `var()` inside fallbacks supported. Legacy alias edges (`--base-unit` etc.) resolve through the graph.
3. **Semantic-derived defaults** evaluated after overlay (expressions see overridden bases).
4. **Parse** — full-match bounded grammar per registry entry; any non-match → registry default + diagnostic, never throw, never partial-match. When the registry default is a semantic-derived EXPRESSION, the fallback RE-EVALUATES it against the post-overlay environment (round-2 fix — never a pre-resolved literal, so an invalid explicit value still respects the consumer's other overrides):
   - color: `rgba()|rgb()|hex(3/4/6/8)|hsl()/hsla()` + `transparent`; channels finite, 0-255 int; alpha finite float clamped [0,1].
   - length: `<finite float>px` (incl. `0px`); bare `0` accepted as 0; other units → fallback + diagnostic.
   - fontWeight: number 1–1000 OR keyword union `normal|bold|lighter|bolder` (mapped to numbers where exact).
   - keyword: per-key closed union from the registry (e.g. textCase: `none|uppercase|lowercase|capitalize`), not "any trimmed string".
   - opacity/number: finite float, per-key range clamp + diagnostic on clamp.
   - shadow: top-level-comma layer split (paren-depth-aware); per layer `[inset] <2-4 lengths> <color>` full-match; `inset` from the layer token only. All layers preserved (`ShadowLayer[]`); RN mapping is 0.7.
   - calc: exactly the SCSS-emitted dialect — `calc(<float> * var(--sjs-base-unit[, <nested var()|literal>]))` including redundant inner parens as emitted; anything else → fallback + diagnostic (documented non-goal: general calc engine).
5. Assemble `ResolvedTheme` (pure data, stable-serializable — deep-equal across identical calls):
   - `tokens`: color groups (parsed `{r,g,b,a}` + normalized string), `baseUnit`/`cornerRadius` numbers, shadows (6 × `ShadowLayer[]`), `articleFont` — **9 props per size incl. derived fontSize**; `typography` — **amended by 0.7's companion amendment 2** (docs/design/0.7-theme-rn.md, "Companion amendments" #2): normalized `base {fontFamily, fontSize}` / `editor {fontFamily, fontSize, fontWeight}` / `questionTitle {fontFamily, fontSize, fontWeight}` families, the two SEPARATE 1.5× line-height tokens (`editorLineHeight`, `baseLineHeight` — identical at defaults, diverge when only one of `--sjs-font-editorfont-size`/`--sjs-font-size` is overridden), and `editorCornerRadius` (`--sjs-editorpanel-cornerRadius`). Family fields resolve through the shared var() engine but treat the documented fallback-less `--sjs-default-font-family` hook as `''` (inherit/RN system default) rather than a diagnostic — the real CSS cascade's answer here is `inherit`, not an error.
   - `meta`: `{ themeName?, colorPalette: 'light'|'dark' (default 'light'), isPanelless: boolean (default false) }`.
   - `background`: `{ image?: raw URI (url() unwrapped), fit: 'auto'|'contain'|'cover' (default 'cover'), attachment: 'scroll'|'fixed' (default 'scroll' — **'fixed' PRESERVED here**; mapping/diagnosing it is 0.7 policy, per review), opacity: finite [0,1] (default 1, clamped + diagnostic) }`.
   - `header`: `{ rawHeader: IHeader | undefined (verbatim), headerView: 'basic'|'advanced' (incl. applyTheme's implicit rule: header present + no explicit headerView ⇒ 'advanced'), backgroundKind: 'none'|'accent'|'custom' (classified BEFORE dereference — `var(--sjs-primary-backcolor)` literal ⇒ 'accent', mirroring Cover), colors: { raw always; resolved evaluated WITH `backgroundKind` as the context selector (accent ⇒ the accent-context fallback chain, else the normal chain — mirroring header.scss; round-2 fix: context-dependent colors are resolvable here precisely because backgroundKind IS the context) } }`.
   - `rawVariables`, `extras` (unknown keys), `diagnostics: ThemeDiagnostic[]` — **returned as data**; the 0.7 provider emits them post-commit through the diagnostics seam with cross-resolution dedup (resolver stays observably pure — round-1 fix).
6. `spacing` is NOT in the output — pure helper `spacing(baseUnit, multiplier)` in `helpers.ts` (no memoization; output stays function-free for deep-equality).

## Test plan (TDD)

1. **Goldens**: one stable serialized snapshot per named preset (40, via themes-facade manifest) + `resolveTheme(undefined)` golden. A legitimate upstream band patch changing a theme value SHOULD fail its named golden — review, re-bless, changelog note (documented workflow, not brittleness).
2. **Default-manifest tests** (presets can't cover these — they shadow all 82 keys): every registry default parses under its own grammar; derived-expression defaults re-evaluate under sparse overrides (`--sjs-font-size` override flows into 5 article fontSizes); **SCSS-parity via independent extraction fixture** (round-2 fix — a golden of the vendored table only proves self-consistency): `scripts/extract-scss-defaults.mjs` parses `var(--sjs-*, …)` fallback chains out of the reference checkout's `default-theme/*.scss`, emits a committed fixture; a test exhaustively compares the registry's defaults against it. Regenerated + re-reviewed on survey-core band bumps (script + fixture both checked in; test fails if fixture and registry drift apart).
3. **Invariance canaries** across 40: 82-key set, baseUnit 8, cornerRadius 4, articleFont preset-props identical.
4. **Parser grammar**: real multi-layer shadow (LayeredDark), inset-in-medium + no-inset-inner (real cases), bare-int alpha, hex/hsl/transparent, fontWeight `bold`, range clamps, full-match rejections (partial garbage → fallback + diagnostic), calc dialect accept/reject pairs.
5. **var() graph**: alias chain > 4 hops (acyclic, valid — locks removal of hop cap), cycle → members invalid + fallback path, nested-fallback parse, legacy `--base-unit` edge, the accent-header real case incl. `backgroundKind: 'accent'` preserved.
6. **Sparse theme**: one overridden base → that token + its derived dependents change, all else === undefined-golden.
7. **Purity**: repeated calls deep-equal; diagnostics identical arrays; no seam calls during resolve (spy). Type-only facade import (loads without survey-core). ESLint fixtures: `react-native` in theme-core fails; `survey-core/themes` outside themes-facade fails.
8. **RN rating contract**: under RN-shaped globals, `applyTheme` + rating question → `updateColors` no-ops (locks review's correction; rating colors documented as 0.7 token consumers).

## Non-goals
