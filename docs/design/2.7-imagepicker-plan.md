## What imagepicker is

A grid of image CHOICE tiles; tap to select. Verified against survey-core
2.5.33: `getTemplate() === "imagepicker"`, `isDefaultRendering() === true`
(template route), value is scalar (single) or ARRAY (`multiSelect: true`).

Key model surfaces:
- `question.multiSelect` (single vs multi), `question.showLabel` (show the
  choice text under the image), `question.isItemSelected(item)` (selection
  state per tile).
- Grid: `getCurrentColCount()` / `responsiveColCount` / `gridColCount`;
  `renderedImageWidth`/`renderedImageHeight` (+ responsive/min/max variants);
  `imageFit` (contain/cover/fill/none — same mapping as ImageQuestion 2.10).
- `question.contentMode` = "image" (v1); "video"/"youtube" are the SAME
  won't-support/deferred as ImageQuestion 2.10 → render nothing + diagnostic.
- Per-choice: `choice.imageLink` + `choice.locImageLink` (localizable),
  `choice.text`, `choice.contentMode`.

## Design

- **Tiles**: iterate `question.visibleChoices`; each tile is a Pressable
  containing an `Image` (loaded exactly like ImageQuestion 2.10 — through the
  central URI policy: `validateUri(rawUri, 'image', policy)`, fail-closed with
  `image-uri-blocked` diagnostic; support `data:` + allowlisted https;
  `resizeMode` from `imageFit`). Optional label (`showLabel`) under the image.
  Size from `renderedImageWidth`/`Height`.
- **Selection**: Single-select: `question.value = choice.value` (re-tap keeps
  it — survey-core 2.5.33 imagepicker has no allowClear). Multi: toggle membership — reuse the
  SAME core path checkbox uses, or `question.value` array manipulation through
  the public setter (prefer a core toggle if one exists; check
  `question.clickItemHandler`/the checkbox base). Selected state via
  `isItemSelected(item)` → a selected border/overlay/checkmark on the tile.
- **Grid layout**: `getCurrentColCount()` columns; a flex-wrap row of
  `${100/colCount}%`-width tiles (same simple grid as the 1.12
  checkbox/radiogroup column note — upstream's exact column-balancing is not
  reproduced; document in DIFFERENCES).
- **Image load failure**: show the choice text (like ImageQuestion 2.10's
  altText fallback) rather than a blank tile.
- **a11y**: each tile is a Pressable with `accessibilityRole` (radio for
  single, checkbox/"imagebutton" for multi) + `accessibilityState.selected/
  checked` + `accessibilityLabel` = choice text.

## Reuse (extract or share with ImageQuestion 2.10)

ImageQuestion (2.10) already has the policy-checked `Image` + resizeMode +
onLoad/onError + `image-uri-blocked` diagnostic. Factor its image-loading piece
into a shared `PolicyImage` component (rawUri + uriConfig + imageFit → a
validated Image with the fail-closed diagnostic + text fallback) that BOTH
ImageQuestion and imagepicker tiles use. This avoids a second copy of the URI-
policy image path. (2.10 is merged/approved — extract carefully, its tests as
the safety net.)
